1. Product Overview

Fortuna is a CBM-compliant brand health tracking platform that measures how strongly a brand is present in category buyers' minds at the moments they enter the category. It is designed for brand strategists, insights teams, and marketing planners.

Unlike traditional brand tracking (awareness scores, likeability), Fortuna measures mental availability — the probability that a buyer will think of your brand when the need to buy arises — using the Category Buyer Memory (CBM) methodology.

The Core Question Fortuna Answers:
"Which brand do buyers think of — and in which buying situations — when they enter this category?"

Who Is It For?

UserPrimary Use
Brand ManagersTrack MPen, MMS, and competitive position wave over wave
Insights / Research TeamsDesign CBM-compliant surveys, upload fieldwork data, review compliance
Media PlannersMeasure effective reach and correct branding for each campaign creative
Strategy ConsultantsUse forecast tools and Mental Advantage maps to build brand growth briefs

2. How the App Works

Current State (v0.5 — UI Build)

Fortuna v0.5 is a fully functional front-end prototype. All dashboards, charts, and analytics pages are built and running with sample/demo data pre-populated. No backend database or authentication is required to run it locally.

Important: The current version uses hard-coded sample data (Indonesian Credit Cards category — BCA as focal brand). A backend database (Phase 1) is planned but not yet implemented. All data entry via modals is UI-only — it does not persist between page refreshes.

Architecture at a Glance

LayerWhat It Does

React Context (SegmentContext)Shares the active segment (Non-buyers+Light / Heavy / Overall) across all dashboard pages globally
Page ComponentsEach analytics page is a self-contained React component with its own data constants and chart logic
RechartsRenders all charts (bar, line, scatter)
DataEntryModalReusable component for manual data entry + Excel upload on every analytics page
CBM Engine (lib/)Frozen formula contracts and compliance validators — not yet wired to live data

User Flow

  1. User lands on http://localhost:3000 — sees the landing page
  2. Clicks Open Dashboard → enters the dashboard at /dashboard
  3. The Segment Toggle (top right of each page) switches between:
    — Non-buyers + Light (default — recruitment focus)
    — Heavy Buyers (retention focus)
    — Overall (competitive benchmarking)
  4. All charts, KPI cards, and analysis text update dynamically when the segment changes
  5. Each analytics page has an Add Data button — opens a modal for manual entry or Excel upload
  6. The Sidebar provides navigation across all 11 pages

Segment System

3. Data Required to Run the App

3.1 To Run Right Now (Demo Mode)

No data input required. The app runs immediately after npm install && npm run dev. All pages are pre-populated with sample data for the Indonesian Credit Cards category (BCA as focal brand, Q4 2024 wave).

3.2 To Use With Real Data (Current Version)

Because there is no database yet, real data must be entered directly into the page source files (app/dashboard/[page]/page.tsx). Each page has data constants at the top that can be replaced with actual wave values.

3.3 Data Required Per Page

Page Data Needed Format
Dashboard Overview MPen %, MMS %, Network Score, SoM/Prompted Awareness per segment per wave. Competitive analysis text. % values per brand per segment per wave
Brand KPIs (Analytics) MPen, MMS, NS, SoM for all brands in category. Both non-light and heavy buyer cuts. % values per brand per segment
Mental Advantage Binary linkage matrix: for each brand × CEP combination, the % of buyers who link them. All brands, all CEPs. Matrix of % values (brands as rows, CEPs as columns)

Physical Availability — Prominence Fame score (% who recognise the asset) and Uniqueness score (% who attribute it only to your brand) for each distinctive asset. Scores 0–100 per asset

WOM Tracker PWOM and NWOM per 100 category buyers for each brand per wave. Binary measurement (yes/no in past month). Count per 100 buyers per brand per wave
Forecast — DJ Forecast Penetration % and purchase frequency per brand per wave. At least 3 brands to fit the DJ curve. % pen + frequency (times/year) per brand

3.5 Data Quality Requirements (CBM Methodology)

RequirementWhy It Matters
Survey format: binary pick-any onlyLikert scales introduce acquiescence bias and inflate all brands equally. Binary forces genuine top-of-mind association.
Brand ordering: alphabeticalPosition bias — brands listed first get higher selection rates. Alphabetical order eliminates this.
No superlatives in CEP wording"Most trusted" biases toward market leaders. Use plain-form: "trusted."
Non-buyer quota ≥30% of sampleNon-buyers are the growth pool. Under-sampling them skews all growth metrics.
Minimum n=500 category buyers per waveBelow this, estimates are unstable. For subgroups, n≥100 required.
Same sample base for all brandsScores must be comparable — different bases cannot be fairly benchmarked.
CEP measured in past 12 months screenerBounded recall ensures only active category buyers are included.

4. Page-by-Page Guide

4.1 Dashboard Overview /dashboard

Segment-aware command centre. Shows 4 KPI cards, a segment analysis panel, and a MMS vs SMS gap diagnosis. The content changes entirely when you switch segments.

  • KPI Cards: MPen, MMS, NS, SoM (or Prompted Awareness) — each with wave-on-wave delta
  • Segment Banner: Active segment + primary KPI + strategic goal label
  • Key Insight: CEO-ready headline + recommendation — unique per segment
  • MMS vs SMS Gap: Diagnoses whether the problem is mental (MMS < SMS) or physical (MMS > SMS)

4.2 Brand KPIs /dashboard/analytics

Competitive benchmarking. All brands ranked by the focal segment's primary KPI. DJ-normalized bar chart.

4.3 Mental Advantage /dashboard/mental-advantage

Brand × CEP heatmap. Each cell shows deviation from DJ-expected score.

Cell LabelDeviationAction
DEFEND≥+5pp above expectedProtect in media — this is a mental asset
MAINTAINWithin ±4ppHold activity — no urgent action
BUILD≤−5pp below expectedTarget in next campaign — mental gap

4.4 Physical Availability /dashboard/physical

Five tabs: Overview · Presence (channel coverage) · Portfolio (SKU contribution) · Prominence (distinctive assets) · Gap Analysis (MMS vs SMS diagnosis per segment).

4.5 Marketing Reach /dashboard/reach

Creative effectiveness tracker. Noticed % + Correct Branding % + Branding Ratio per creative per wave. Below 60% Branding Ratio = "vampire creative" flag.

4.6 WOM Tracker /dashboard/wom

PWOM and NWOM per 100 category buyers. Net WOM = PWOM − NWOM. WOM Ratio = PWOM ÷ NWOM (>4× is strong).

4.7 Forecast /dashboard/forecast

Five forecast methodologies in tabs:

TabWhat It Does

4.8 CEP Builder /dashboard/ceps

Build a CBM-compliant attribute list using the 7W framework. Composition meter enforces 60–70% CEPs. CBM Engine flags wording violations in real time.

4.9 Surveys /dashboard/surveys

Block-based survey canvas. 17 question blocks in the library. Core blocks (mandatory) cannot be removed. Upload Results tab accepts Excel fieldwork data.

5. CBM Methodology Reference

The CBM Mantra

#PrincipleWhat It Means in Practice
1Design for the CategoryCEPs belong to the category, not the brand. Every brand is measured against the same buying occasions.
2Analyse for the BuyerNon-buyers + light buyers are the default, primary segment. They are the growth pool.
3Report for the BrandScores are DJ-normalized — small and large brands are fairly compared by deviation from expected.

Double Jeopardy (DJ) Law

Brands with lower penetration suffer double — they have fewer buyers AND those buyers buy them less frequently. This is a market law, not a strategy. Brands naturally cluster along the DJ curve.

  • Above the DJ curve: structural advantage (unusual)
  • On the DJ curve: performance matches brand size
  • Below the DJ curve: loyalty problem — buyers leaving faster than expected

Category Entry Points (CEPs)

A CEP is a situational trigger that activates category buying. CEPs are buying occasions — not brand attributes. They are derived using the 7W Framework:

WQuestionExample CEP (Credit Cards)
WhoWho is the buyer?"When paying for my family's expenses"
WhatWhat is the occasion?"When making an online purchase"
WhenWhen does the need arise?"When travelling internationally"
WhereWhere does it happen?"When checking out at a restaurant"
WhyWhat triggers it?"When I need emergency cash"
With WhatWhat tool is used?"When tapping to pay contactlessly"
With WhomWho is involved?"When splitting a bill with friends"

MMS vs SMS Gap — Diagnosis Framework

ConditionMeaningRecommended Action
MMS > SMS (positive gap)Physical availability problem — buyers think of the brand but can't find or buy itExpand distribution. Close channel coverage gaps.
MMS < SMS (negative gap)Mental availability problem — sales driven by habit, mental links are fragileReinforce CEP coverage in media. Build distinctive assets.
MMS ≈ SMS (gap < ±3pp)Healthy parityMaintain current investment mix.

7. Setup & Running Locally

Step 1 — Install Node.js

Download and install Node.js 18 or higher from https://nodejs.org. The LTS version is recommended.

Verify after installation:

node --version   # should show v18.x.x or higher
npm --version    # should show 9.x.x or higher

Step 2 — Clone the Repository

git clone https://github.com/matiyashu/Brand-Health-Tracker---Mental-Availability-Physical-Availability.git
cd Brand-Health-Tracker---Mental-Availability-Physical-Availability

Step 3 — Install Dependencies

npm install

This reads package.json and installs all required packages into the node_modules/ folder. Takes 30–60 seconds on first run.

Step 4 — Run the Development Server

npm run dev

Open your browser and go to: http://localhost:3000

No environment variables needed. The app runs in demo mode with sample data. No database connection string, no API keys, no .env file required for the current UI build.

Step 5 — Build for Production (Optional)

npm run build   # creates optimised production build
npm start       # serves the production build

8. Formula Reference

KPIFormulaTarget / Benchmark
Mental Penetration (MPen) Buyers linking ≥1 CEP to brand ÷ Total sample × 100 >50% is strong for a market leader
Mental Market Share (MMS) Brand's total CEP links ÷ Total CEP links (all brands) × 100 Should track close to Sales Market Share
Network Score (NS) Total CEP links for brand ÷ Buyers with MPen for that brand >4.0 is healthy breadth
Share of Mind (SoM) Brand CEP links ÷ Total links among MPen base × 100 Benchmark against MMS
DJ Expected Score (Row Total × Column Total) ÷ Grand Total Used to calculate Mental Advantage deviation
Mental Advantage Deviation Actual Score − DJ Expected Score ≥+5pp = DEFEND · ±4pp = MAINTAIN · ≤−5pp = BUILD
Net WOM PWOM − NWOM Positive is healthy; negative is urgent
WOM Ratio PWOM ÷ NWOM >4× is strong advocacy balance
Branding Ratio (Reach) Correctly Branded % ÷ Noticed % >70% is strong; <60% = "vampire creative"
MMS → SMS Correlation r = 0.83 (MMS leads SMS by 1–2 quarters) MMS > SMS = physical gap; SMS > MMS = mental gap

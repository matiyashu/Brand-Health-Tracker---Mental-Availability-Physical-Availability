# Brand Health Tracking App — Project Context

## Product Overview
CBM-compliant brand health tracker for Digital& agency.
Based on Better Brand Health (Ehrenberg-Bass principles).

## Brand Identity
- Dark charcoal: #1A1A1A
- Neon green: #0FA958
- BCG-style outputs: CEO headline, So What / Now What

## CBM Methodology Rules (NON-NEGOTIABLE)

### Formula Contracts
Files in /lib/cbm-engine/formula-contracts/ are FROZEN.
DO NOT modify their logic under any circumstances.

### Survey Design Rules
- NEVER add Likert scales or numeric scales
- NEVER allow non-alphabetical brand ordering
- NEVER use modified attribute forms (most X, best X, more X)
- Binary pick-any is the ONLY accepted response format

### Attribute Composition
- 60-70% CEPs (Category Entry Points via 7W framework)
- ≤30% baseline competencies
- ≤20% secondary attributes

### Analysis Defaults
- Default view = non-buyers + light buyers
- Always show DJ-normalized competitive comparisons
- Auto-flag MMS/SMS gaps

## The CBM Mantra
1. Design for the Category
2. Analyze for the Buyer (non-buyers + light buyers are primary)
3. Report for the Brand (control for size via DJ)
